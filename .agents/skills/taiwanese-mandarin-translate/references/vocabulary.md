# Taiwan Mandarin vocabulary reference

Use this file as a quick terminology reference when translating UI copy, documentation, and software-related text into Traditional Chinese for Taiwan readers.

## Register guidance

| Situation | Default choice |
| --- | --- |
| everyday product and documentation writing | 台灣華語優先，求自然、清楚、順口 |
| broad public-facing or formal text | 在台灣華語基礎上，往中性、標準的中華民國國語書面語靠攏 |
| government or education context mentioning Taiwan | 優先留意「臺／台」與正式語體的一致性 |
| pronunciation or language-learning content | 優先採用台灣教育體系常見的術語與表示法 |

## Context-sensitive software and product terms

Some English terms should not be mapped to a single Chinese word without context.

| English | Prefer in Taiwan | Notes |
| --- | --- | --- |
| support | 支援 / 客服 / 支持 | Use `支援` for technical capability, `客服` for customer support teams, and `支持` only for endorsement or advocacy. |
| issue | 問題 / 議題 / issue | Use `問題` for bugs or user-facing problems, `議題` for policy or discussion topics, and keep `issue` for GitHub-style work items when appropriate. |
| account | 帳戶 / 帳號 | Use `帳戶` for a user account as a whole; use `帳號` for an identifier or login name. |
| service | 服務 / 服務項目 | Usually `服務`; avoid PRC-style overuse of `業務`. |
| session | 工作階段 / session | Use `工作階段` for user sessions; keep `session` if it is a technical protocol or product term. |
| token | token / 權杖 | Keep `token` in developer-facing docs unless the project already uses `權杖`. |
| submit | 送出 / 提交 | Use `送出` for forms and UI; `提交` is acceptable in formal review or application contexts. |
| deploy / deployment | 部署 | Standard Taiwan software term. |
| sync | 同步 | Avoid `同步化` unless the project uses it. |
| plan | 方案 / 計畫 / plan | Use `方案` for pricing/product tiers, `計畫` for schedules or initiatives, and keep `plan` if it is a product label. |

## Style reminders

- Prefer concise, natural Taiwan Mandarin over literal translation.
- Treat Taiwanese Mandarin as the default voice, and use ROC Mandarin mainly as a neutral written register for formal Taiwan contexts.
- Use full-width punctuation in prose.
- Preserve English for brands, product names, APIs, commands, code, file paths, and placeholders unless asked otherwise.
- Keep terminology consistent across the whole output.
- Preserve the project's established terminology and `台／臺` convention unless the user asks for normalization.
